# v0.1 Acceptance Report Format

## Contract: Acceptance report structure

The v0.1 acceptance report SHALL include the following sections:

```json
{
  "reportVersion": "0.1",
  "timestamp": "ISO-8601",
  "owl4agentsVersion": "string — build or commit hash",
  "fixtures": {
    "golden": [
      {
        "fixtureId": "string — fixture name, e.g. '01-minimal'",
        "file": "string — relative path to fixture file",
        "status": "pass | fail | skip",
        "scenarios": [
          {
            "scenarioName": "string — e.g. 'import_valid_ontology'",
            "gate": "string — e.g. 'Import', 'Summary', 'Class context'",
            "status": "pass | fail | skip",
            "actualOutput": { ... },
            "expectedContract": "string — contract reference path",
            "differences": []
          }
        ]
      }
    ],
    "smoke": [ ... ],
    "benchmarks": [ ... ]
  },
  "cliMcpParity": {
    "checks": [
      {
        "operation": "string — e.g. 'summary', 'search'",
        "fixtureId": "string",
        "cliOutput": { ... },
        "mcpOutput": { ... },
        "equivalent": "boolean",
        "differences": []
      }
    ],
    "summary": {
      "totalChecks": "integer",
      "passed": "integer",
      "failed": "integer"
    }
  },
  "readonlySafety": {
    "checks": [
      {
        "scenario": "string — e.g. 'reject_sparql_update'",
        "interface": "cli | mcp | sparql",
        "errorCodeExpected": "string",
        "errorCodeActual": "string",
        "passed": "boolean",
        "workspaceUntouched": "boolean"
      }
    ],
    "summary": {
      "totalChecks": "integer",
      "passed": "integer",
      "failed": "integer"
    }
  },
  "overall": {
    "totalScenarios": "integer",
    "passed": "integer",
    "failed": "integer",
    "skipped": "integer",
    "acceptanceStatus": "pass | fail",
    "failureDetails": [
      {
        "fixtureId": "string",
        "scenarioName": "string",
        "gate": "string",
        "reason": "string"
      }
    ]
  }
}
```

## Acceptance Status Rules

The overall acceptance status SHALL be:
- `"pass"` if ALL required scenarios pass AND CLI/MCP parity passes AND readonly safety checks pass
- `"fail"` if any required scenario fails, parity check fails, or safety check fails

Required scenarios are defined per the coverage matrix in the corpus manifest.

## Report Generation

The acceptance report SHALL be generated by running the full v0.1 acceptance suite and collecting all scenario results. It SHALL be written as a JSON file to the workspace or a specified output path.
