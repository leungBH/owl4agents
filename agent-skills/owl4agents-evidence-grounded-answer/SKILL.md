# owl4agents Evidence-Grounded Answer Skill

Standard operating procedure for writing answers grounded in ontology evidence using the owl4agents v0.5 workflow.

## Purpose

This skill enables LLM agents to write answers that are grounded in ontology evidence, using the evidence context generated by the v0.5 workflow. The evidence context provides a budget-aware, compact summary of verification results suitable for injection into agent prompts.

## Prerequisites

- A completed claim verification report (from `ontology_verify_claims_batch` or `ontology_review_answer_claims`)
- An evidence context (from `ontology_build_evidence_context` or `ontology_review_answer_claims`)
- Familiarity with the [verdict-policy.md](../_shared/references/verdict-policy.md) and [evidence-citation-policy.md](../_shared/references/evidence-citation-policy.md)

## Standard Operating Procedure

### Step 1: Obtain Evidence Context

Use `ontology_build_evidence_context` or `ontology_review_answer_claims` to generate a compact evidence context:

**MCP tool (with policy guidance):**
```json
{
  "ontology_id": "pizza-ontology",
  "claims": { "answerId": "answer-001", "claims": [...] },
  "max_context_tokens": 500,
  "policy": "strict"
}
```

**CLI command:**
```bash
owl4agents review-answer pizza-ontology --claims test/fixtures/v0.5/answer-claims-supported.json --max-context-tokens 500 --policy strict --json
```

The response includes:
- `report`: full verification report
- `evidenceContext`: compact context with claims, evidence, and agent instructions
- `policy`: the review policy used
- `handlingGuidance`: policy-dependent instructions

### Step 2: Follow Agent Instructions

The evidence context includes mandatory `agentInstructions`:
- "Cite only evidence returned in this context."
- Status-dependent instructions (varies by aggregate status and policy)

Follow these instructions and the [evidence-citation-policy.md](../_shared/references/evidence-citation-policy.md). Do not supplement with external knowledge or fabricated evidence.

### Step 3: Write the Answer

Structure your answer based on the evidence context:

1. **For verified claims (`ENTAILED`):** Present as supported facts, citing the evidence entries.
2. **For contradicted claims (`NOT_ENTAILED`):** State that the claim is contradicted, citing counterexamples.
3. **For unknown claims (`UNKNOWN`):** State that evidence is insufficient. Do not guess.
4. **For out-of-scope claims:** State that the ontology does not cover the referenced entities. Do not make claims about entities outside the ontology's domain.

### Step 4: Handle Truncation

If `omittedClaimCount` > 0 or `omittedEvidenceCount` > 0:
- State that additional evidence exists but was omitted due to budget constraints.
- Do not reference omitted evidence as if it were available.
- Do not fabricate what the omitted evidence might have shown.

### Step 5: Apply Review Policy

Follow the `handlingGuidance` returned by the review:

- **strict**: Do not present any claim as fact unless it is supported by ontology evidence. A contradicted required claim means the answer must be rejected.
- **conservative**: Only cite explicitly verified claims. Verify optional claims independently before citing.
- **report-only**: Present the factual findings without judgment. The user decides how to interpret the evidence.

## Example

Using the `pizza-ontology` fixture with `max_context_tokens=500` and `policy=strict`:

```bash
owl4agents review-answer pizza-ontology --claims test/fixtures/v0.5/answer-claims-supported.json --max-context-tokens 500 --policy strict --json
```

The response provides:
- `aggregateStatus`: "verified" if all required claims are supported
- `handlingGuidance`: "Policy: strict — do not present any claim as fact unless it is supported by ontology evidence."
- `agentInstructions`: "Cite only evidence returned in this context."

## Important Notes

- The evidence context never fabricates evidence. If no evidence is available, evidence lists are empty.
- Under truncation, all claim IDs and verdicts remain visible. Only evidence entries are omitted.
- The `4 * maxContextTokens` character budget controls truncation. Set `max_context_tokens=0` for no truncation.
- Do not present claims as verified unless they have `ENTAILED` verdict with supporting evidence.
- See [verdict-policy.md](../_shared/references/verdict-policy.md) for verdict interpretation.
